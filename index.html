القسم الأول · Control Flow

إشارات المرور في كودك: التحكم في مسار التنفيذ

كما تنظّم إشارات المرور حركة السيارات، تنظم جمل التحكم مسار الكود — متى يتوقف، متى ينعطف، ومتى يكرر نفسه.

 لماذا "إشارات المرور"؟

تخيّل كودك كسيارة تسير في طريق. أحيانًا يجب أن تتوقف عند شرط (if)، وأحيانًا تدور في حلقة دائرية (for / while) حتى تصل إلى وجهتها. بدون هذه الإشارات، ينفّذ بايثون سطورك سطرًا بعد سطر بلا أي انعطاف — ومعها يصبح الكود ذكيًا قادرًا على اتخاذ القرارات والتكرار.

  • الشرط (if): قرّر هل تنفّذ كتلة كود بناءً على شرط منطقي.
  • الحلقة (for): كرّر تنفيذ كود بعدد محدود من المرات.
  • الحلقة (while): استمر في التنفيذ طالما الشرط صحيح.

الجمل الشرطية if

إشارة حمراء: توقف الكود واسأل — هل الشرط صحيح؟ إن نعم، خذ هذا المسار.

example.py
if score >= 90:
    grade = 'A'
elif score >= 80:
    grade = 'B'
else:
    grade = 'C'

نصيحة: يمكن دمج الشروط بـ and و or. تجنّب تداخل أكثر من 3 مستويات — استخدم دالة بدلًا من ذلك.

حلقة for

إشارة صفراء: كرّر التنفيذ عبر عناصر قائمة أو نطاق محدد — تعرف بدايتك ونهايتك.

example.py
# طباعة الأرقام من 0 إلى 4
for i in range(5):
    print(i)

# المرور على عناصر قائمة
for name in names:
    print(name)

نصيحة: استخدم enumerate() للحصول على الفهرس والعنصر معًا بدلًا من range(len(...)).

حلقة while

إشارة خضراء: استمر في الدوران طالما الشرط صحيح — مثالية للحالات غير المحدودة مسبقًا.

example.py
count = 0
while count < 5:
    print(count)
    count += 1

# انتبه: حدّث المتغير
# وإلا ستدور للأبد!

تحذير: نسيت تحديث المتغير؟ حلقة لا نهائية. استخدم break للخروج الطارئ.

خزائن البيانات: Lists و Sets

البيانات تحتاج أن تُخزَّن بذكاء. القوائم تحفظ الترتيب وتقبل التكرار، والمجموعات تضمن التفرد وتعمل بسرعة البرق.

 لماذا "الخزائن"؟

القائمة list كخزانة بأدراج مرقّمة — كل عنصر له موقع محدد، ويمكن تكرار المحتوى. المجموعة set كصندوق واحد يجمع العناصر الفريدة فقط — لا تكرار، ولا ترتيب، لكن البحث فيها فائق السرعة.

  • القائمة List: مرتبة، قابلة للتعديل، تقبل التكرار — للمجموعات المتسلسلة.
  • المجموعة Set: غير مرتبة، فريدة — لإزالة التكرار والبحث السريع.

القوائم Lists

fruits = ['apple', 'banana', 'apple']

خزانة بأدراج مرقّمة — تحفظ الترتيب وتقبل التكرار. مثالية للبيانات المتسلسلة.

مرتبة (تحافظ على ترتيب الإدراج)
تقبل العناصر المكررة
قابلة للتعديل (إضافة/حذف)
الوصول بالفهرس O(1)
البحث عن قيمة O(n) — بطيء
fruits = ['apple', 'banana', 'apple']
print(fruits[0])       # 'apple'
print(len(fruits))    # 3
fruits.append('orange')
fruits.remove('banana')

المجموعات Sets

unique = {'apple', 'banana', 'orange'}

صندوق العناصر الفريدة — لا تكرار ولا ترتيب، لكن البحث والإضافة بسرعة البرق.

عناصر فريدة فقط (بلا تكرار)
البحث والإضافة والحذف O(1)
عمليات تركيبية (اتحاد، تقاطع)
غير مرتبة — لا وصول بالفهرس
لا تقبل العناصر القابلة للتغيير
unique = {'apple', 'banana', 'apple'}
print(unique)        # {'apple','banana'}
print('apple' in unique)  # True · سريع!

a = {1,2,3}; b = {3,4,5}
print(a & b)          # {3}  تقاطع
print(a | b)          # {1,2,3,4,5} اتحاد
القسم الثالث · Pro Trick

حيلة المحترفين: List Comprehension

لماذا يكتب المحترفون حلقة for في سطر واحد؟ لأنها أسرع، أنظف، وأكثر "بايثونية". دعنا نقارن.

الطريقة التقليدية ❌
slow.py
squares = []
for i in range(1000):
    squares.append(i ** 2)
المشكلة: استدعاء .append() ألف مرة + إنشاء قائمة فارغة + حلقة صريحة = كود طويل وأبطأ.
طريقة المحترفين ✅
fast.py
squares = [i ** 2 for i in range(1000)]
الميزة: سطر واحد، أسرع بنسبة ~50–70%، ومنطقها مضمن في C داخل بايثون.

مقارنة سرعة التنفيذ

حساب مربعات الأرقام من 0 إلى 999,999 — متوسط 100 تشغيل (microseconds).

for + append
List Comprehension
map() + lambda
~2.1x
أسرع مع List Comprehension مقارنة بـ for التقليدية
1 سطر
بدلًا من 3 أسطر — كود أنظف وأقصر
C-level
التنفيذ يتم في طبقة C داخل بايثون — لا overhead

 صيغ متقدمة لـ List Comprehension 

# 1. مع شرط (filter)
evens = [x for x in range(20) if x % 2 == 0]

# 2. مع شرط if/else (transform)
labels = ['even' if x%2==0 else 'odd' for x in range(10)]

# 3. حلقات متداخلة
pairs = [(x,y) for x in range(3) for y in range(3)]

# 4. Dict comprehension
sq_map = {x: x**2 for x in range(5)}
تبويبات تفاعلية

شرح للمبتدئين · ونصائح للمحترفين

اختر مستواك — المبتدئ يفهم الأساسيات، والمحترف يكتشف أسرار الأداء.

ابدأ من هنا — المفاهيم الأساسية

  • التحكم في التدفق يعني قدرتك على جعل الكود يتخذ قرارات: "إذا كان كذا، افعل كذا". هذا ما يجعل البرامج ذكية.
  • الجملة الشرطية if تسأل سؤالًا نعم/لا. إن كان الجواب "نعم"، تُنفّذ الكود بداخلها.
  • الحلقة for تكرر كودًا على مجموعة عناصر — مثل المرور على كل طالب في قائمة وتحيّته.
  • الحلقة while تكرر الكود "طالما" شرط معين صحيح — مثل: استمر في القرع على الباب طالما لم يفتح أحد.
  • القائمة list وعاء يحفظ عدة عناصر بالترتيب — مثل قائمة التسوّق.
  • المجموعة set وعاء يحفظ عناصر فريدة فقط — مثل مجموعة أصدقائك (لا يمكن تكرار نفس الشخص).
  • List Comprehension اختصار جميل لكتابة حلقة for في سطر واحد لإنشاء قائمة — ستتعلمها بالممارسة.

ارفع مستواك — أسرار الأداء والأناقة

  • استبدل range(len(x)) بـ enumerate(x) — أنظف وأسرع ويعطيك الفهرس والعنصر معًا.
  • للبحث في مجموعات كبيرة، حوّل القائمة إلى set أولًا — البحث O(1) بدلًا من O(n). فرق هائل على مليون عنصر.
  • تجنّب تعديل القائمة أثناء المرور عليها — يسبب أخطاء خفية. أنشئ نسخة أو استخدم list comprehension بدلًا من ذلك.
  • استخدم any() و all() للتحقق من وجود/شمول شرط — أسرع وأوضح من حلقة for.
  • Profile قبل التحسين — استخدم cProfile و timeit لمعرفة الجزء البطيء فعلًا. لا تخمين.
  • المولدات (x for x in ...) أفضل من List Comprehension للبيانات الضخمة — لا تستهلك ذاكرة.
  • احذر: List Comprehension المفرط — إن تجاوزت 80 حرفًا أو تداخلت حلقتان، عُد للحلقة العادية. الوضوح أهم من الإيجاز.
أخطاء شائعة

أخطاء يقع فيها الجميع — وحلولها

انقر على كل خطأ لكشف المشكلة والحل. هذه أكثر الأخطاء تكرارًا عند تعلّم التحكم في التدفق وهياكل البيانات.

01
حلقة while لا نهائية بسبب نسيان تحديث المتغير
المشكلة: نسيت زيادة المتغير داخل الحلقة، فبقي الشرط صحيحًا للأبد، وتجمّد البرنامج. 
count = 0
while count < 5:
    print(count)  # تكرار لا نهائي!
الحل: حدّث المتغير داخل الحلقة دائمًا، أو استخدم for مع range() حين يكون عدد التكرارات معروفًا. 
count = 0
while count < 5:
    print(count)
    count += 1  # ✅ التحديث
02
الخلط بين = و == في الشروط
المشكلة: استخدام = (إسناد) داخل if بدلًا من == (مقارنة) — يسبب خطأ نحوي أو منطقي خفي. 
if x = 5:  # ❌ SyntaxError
    print('x is 5')
الحل: استخدم == للمقارنة، و = للإسناد فقط. (ملاحظة: بايثون 3.8+ يدعم := كعامل حمار، لكنه متقدم.) 
if x == 5:  # ✅ صحيح
    print('x is 5')
03
استخدام list للتحقق من وجود عنصر في بيانات ضخمة
المشكلة: البحث في قائمة كبير باستخدام in يأخذ وقتًا خطيًا O(n) — كارثة على ملايين العناصر. 
emails = ['a@x.com', 'b@x.com', ...]  # مليون عنصر
if 'z@x.com' in emails:  # ❌ بطيء
    ...
الحل: حوّل القائمة إلى set مرة واحدة، ثم ابحث فيها — البحث O(1) فائق السرعة. 
email_set = set(emails)  # مرة واحدة
if 'z@x.com' in email_set:  # ✅ سريع جدًا
    ...
04
تعديل قائمة أثناء المرور عليها بحلقة for
المشكلة: حذف أو إضافة عناصر أثناء التكرار يسبب تخطّي عناصر أو سلوك غير متوقع. 
for item in items:
    if item < 0:
        items.remove(item)  # ❌ خطر
الحل: استخدم List Comprehension لإنشاء قائمة جديدة، أو كرّر على نسخة items[:]. 
items = [x for x in items if x >= 0]  # ✅ آمن وأنظف
05
إساءة استخدام List Comprehension لكل شيء
المشكلة: بعض المبرمجين يستخدم List Comprehension لأسباب جانبية (مثل الطباعة) أو بحلقات متداخلة معقدة، فيصبح الكود غير مقروء. 
[print(x) for x in items]  # ❌ يستخدم لغير هدفه
result = [a+b for a in A for b in B if a>b for c in C if c==a]  # ❌ معقد جدًا
الحل: List Comprehension لإنشاء قائمة بمنطق واضح. للطباعة استخدم حلقة عادية. للتداخل المعقد، حلقة for صريحة أو دالة منفصلة. 
# ✅ استخدام صحيح
squares = [x**2 for x in range(10)]

# ✅ للطباعة — حلقة عادية
for x in items:
    print(x)

البرمجة ليست حفظًا — بل تفكيرًا منظّمًا

تعلّمت كيف توجّه كودك بإشارات التحكم، كيف تخزّن بياناتك بخزائن ذكية، وكيف تكتب كودًا أسرع وأكثر أناقة بحيلة المحترفين. الآن حان وقت التطبيق — فالإتقان يأتي منtrial والأخطاء.
